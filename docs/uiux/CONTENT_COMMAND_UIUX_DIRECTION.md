# Content Command UI/UX Direction

Use this as the product design north star for making Content Command feel premium, simple, and sellable.

Reference inspiration:
https://medium.com/@theymakedesign/web-app-design-examples-vol-233-946b304e8b11

## Product Feel

- Sleek, dark, focused, premium.
- Feels like an operator command center, not a generic AI chat tool.
- Every page should make the next action obvious in 5 seconds.
- Visual polish should support workflow speed, not become decoration.

## Design Rules

- One primary action per screen.
- Use the same card radius, border, spacing, button states, and typography rhythm everywhere.
- Keep the palette restrained: dark base, white text, muted gray body copy, cyan for system intelligence, purple for active agent actions, green for passed status, amber for warnings.
- Use icons to reduce reading, but pair icons with text for important actions.
- Show status clearly: draft, ready, running, passed, failed, copied, saved.
- Avoid large decorative sections unless they explain the product faster.
- Avoid mystery panels. If a section exists, the user should know what it does immediately.
- Keep agent pages action-first: mission, run, output, send forward.
- Keep client pages business-first: workspace, voice, proof, offers, saved outputs.
- Keep sales pages execution-first: target, lead list, message, follow-up, close.

## Agent Studio Direction

Agent Studio should answer four questions instantly:

1. What do I give the agents?
2. Which agent or pod do I run?
3. What did they produce?
4. What do I do with the output?

Current ideal layout:

- Mission Brief at the top.
- Primary buttons: Run selected agent, Run pod, Run full swarm.
- Output console next to the brief.
- Agent picker below.
- Selected agent detail panel on the side.
- Run history as a utility, not the main experience.

## Generate Page Direction

Generate should feel like the money page.

- Default mode: Paste Content.
- Make source intelligence visible before generation.
- Let user approve the brief before the agents run.
- Show pipeline progress clearly.
- Output should feel packaged: long post, LinkedIn, X, IG, carousel, script, email, blog, lead magnet, client handoff.
- Each output needs copy, edit, save, and quality state.

## Enterprise Sellability

To sell this to companies, the UI needs:

- Client workspace switcher.
- Brand memory per client.
- Proof and source tracking.
- Quality gate with clear failure reasons.
- Export or client handoff view.
- Performance learning loop.
- Team-safe language: what ran, what passed, what needs review.
